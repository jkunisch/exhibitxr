## Vision

**ExhibitXR / 3D-SNAP** ist ein modernes, performantes **3D-Studio für Creator, Builder und moderne Brands**. Nutzer verwandeln Produktfotos in Sekunden in hochwertige 3D-Assets, die nativ im Web und in AR (iOS & Android) erlebt werden können.

**Kern-Flow:** Foto uploaden → Meshy AI generiert 3D → GLB Optimizer → **Natives AR (USDZ für iOS / GLB für Android)** → Instant Sharing.

## Stack

| Layer | Technologie |
|-------|-------------|
| Framework | Next.js 16 (App Router) |
| Sprache | TypeScript strict, kein `any` |
| UI | React 19, TailwindCSS 4, Framer Motion |
| 3D | React Three Fiber v9 RC, @react-three/drei v10 RC, Three.js |
| State | Zustand 5 |

| AI (3D) | Meshy Pro API (Image-to-3D, PBR, GLB) |

## Architektur (Auszug)

- `src/app/actions/auth.ts`: Handling von Studio-Registrierung (Google Auth) und Session.
- `src/app/actions/generate3d.ts`: Pipeline für 3D-Generierung, Optimierung und AR-Bereitstellung.
- `src/components/ui/ModelGeneratorPanel.tsx`: Das Herzstück des Studios (Upload & Progress).
- `src/lib/glbOptimizer.ts`: Wichtig für Ladezeiten (Ziel: < 3MB).

## Strikte Regeln (Modern Style)

1. **`schema.ts` ist heilig** — NICHT ändern ohne explizite Erlaubnis.
2. **TypeScript strict** — kein `any`.
3. **Studio-Modell** — "Tenant" ist ein internes Konzept; nach außen kommunizieren wir "Studio". Daten liegen unter `/tenants/{tenantId}/`.
4. **Mobile-First** — Das Studio muss auf dem Smartphone "geil" aussehen und funktionieren.
5. **Omnichannel AR** — Jedes Modell muss als GLB (Android/Web) und USDZ (iOS) verfügbar sein.
6. **Keine Bürokratie** — UI/UX muss clean und schnell sein (Apple-Vibe), keine unnötigen Formularfelder oder Textwüsten.
