## Vision

**ExhibitXR** ist ein B2B SaaS, das Firmen ermöglicht, aus Produktfotos automatisch
realistische 3D-Modelle zu generieren und als interaktive Web-Showrooms einzubetten.

**Kern-Flow:** Foto hochladen → Meshy AI generiert 3D → GLB Optimizer (40MB → 2-3MB)
→ Firebase Storage → Embeddable Viewer auf jeder Website.

## Strikte Regeln

1. **`schema.ts` ist heilig** — NICHT ändern ohne explizite Erlaubnis
2. **TypeScript strict** — kein `any`, kein `as unknown as X`
3. **Server Actions statt API Routes** wo möglich
4. **Multi-Tenancy** — alle Daten unter `/tenants/{tenantId}/`
5. **Security**: `request.auth.token.tenantId == tenantId` in Firestore Rules
6. **Edge Middleware** — nur Cookie-Gate, kein `firebase-admin`
7. **Storage** — tenant-protected, `isPublished` Metadata für public reads
8. **3D Components** — immer `'use client'`
9. **`useFrame`** — nur `getState()`, nie `setState`
10. **Commits** — `feat:`, `fix:`, `chore:` Format
