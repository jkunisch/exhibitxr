# Feature Roadmap: Post-Generation & Studio Enhancements (2026)

## A) Low-Hanging Fruits
- **Mesh “Healing” (Auto-Repair):** Automatically fix holes, non-manifold edges and stray fragments using existing tools. For example, Open3D’s `fill_holes()` or MeshFix algorithms can fill small gaps and remove loose pieces【18†L1-L9】. We can wrap these in a serverless function to auto-clean models. *Feasibility:* Medium – run Python-based libraries (Open3D/PyMeshFix) in Firebase functions or a Docker microservice, callable from Next.js.  
- **Basic Retopology (Decimation):** Provide a quick mesh simplification or “garbage collector” that re-triangulates dense meshes to a set polygon count (e.g. via quadric decimation). This improves performance with minimal user effort. Many JS/WebGL libraries or even open-source C++ tools (e.g. Instant Meshes) can be run offline to reduce polycount. *Feasibility:* Easy – can use a WebAssembly decimator or call an API, then serve lighter GLB/USDZ.  
- **Texture Enhancement (Upscale & Denoise):** Use AI upscalers (e.g. Real-ESRGAN) to raise texture maps to 4K/8K and remove noise. Simple filters can also adjust exposure or contrast. This could be exposed as a “High-Res Texture” button. For example, Tripo AI already outputs full PBR maps (normal, albedo, roughness, etc.)【49†L275-L283】, so similar upscaling is feasible. *Feasibility:* Medium – wrap a small image AI model in a cloud function, or call an image-upscaling API.  
- **Quick Style Filters:** Offer a small library of one-click material presets (e.g. change leather to matte/silk, plastic to metal) by swapping or tweaking PBR parameters. This is a precursor to full style-transfer. *Feasibility:* Easy – apply simple material swaps or tint shaders in the viewer (no complex AI needed).  
- **Scene Staging Basics:** Add a flat ground plane and a simple backdrop (solid color or basic HDRI) with one directional light. Turn on SSAO (ambient occlusion) for subtle contact shadows【40†L58-L66】. Enable a “turntable” animation that slowly rotates the model on a pedestal. *Feasibility:* Easy – these are built-in in three.js or Babylon.js (ground planes, SSAO post-process, keyframe rotate). No heavy compute, just UI toggles.  

## B) Mid-Term Differentiators
【45†embed_image】 *AI-Generated HDRI Environments:* Integrate a text/image-driven HDRI generator (e.g. NVIDIA’s Picasso) to create custom 360° environment maps on demand【38†L54-L58】. For instance, a user prompt “showroom garage at sunset” yields a photoreal 8K HDRI background. This instantly gives each product a tailored, high-end studio setting. *Feasibility:* Moderate – call a cloud AI API (serverless via Firebase) to generate or fetch HDRI; store it for IBL in the viewer. No frontend issue in Next.js, but may need GPU-backed service.  
- **Advanced Style/Material Transfer:** Offer true neural style-transfer for 3D meshes. Research prototypes (e.g. Text2Mesh 2022) already use text prompts to recolor a mesh’s vertices or bump map【50†L1814-L1822】. By 2026 we expect services that can “paint” a snapped model in a new style (cyberpunk, cartoon, velvet fabric, etc.) via a one-click or text prompt. *Feasibility:* Challenging – likely require calling a specialized AI service or new API. Could be implemented as a premium backend task (e.g. call a PyTorch model or external API).  
- **AI Auto-Rigging & Posing:** Add an optional “auto-rig” for characters or articulated products. Modern tools (Mixamo, 3D-AI Studio) analyze a humanoid mesh and automatically add a bone structure + skin weights【42†L37-L45】【42†L43-L49】. We can offer a one-click rigging for supported types (e.g. T-posed humanoids or four-legged pets) and provide some preset poses or animations (idle, walk-cycle, turntable). *Feasibility:* Moderate – call an auto-rig API (server-side) on uploaded GLB; feed output rig into viewer. Next.js can trigger this and save the rigged model.  
- **Game-Ready Retopology (Quad Remesh):** Introduce a premium “Pro Topology” service that converts a messy scan into a clean quad mesh suitable for game engines. Providers like Tripo AI already offer “smart retopology” via API【49†L275-L283】. We can use a similar API or open-source Blender headless (QuadRemesh) to retopologize. *Feasibility:* Medium – heavy compute (may need Node.js spawning Blender/Instant Meshes in Firebase Functions or external service). It fits a paid add-on (e.g. +2 credits) since it’s resource-intensive.  
- **Physical Animations:** Enable simple physics- or keyframed animations: e.g. dropping the model onto a table, or a looping spinning turntable. Or if rigged, a simple idle motion. Also allow users to adjust camera moves (dolly, orbit). *Feasibility:* Easy/Medium – implementable with existing physics engines (Cannon.js, Ammo) or custom scripts in the viewer. Next.js just serves the configuration.  

## C) Moonshot (“High-End 2026 Tech”)
- **Full Scene Builder via AI:** In the future, let users describe a scene (text prompt or rough sketch), and auto-generate a 3D studio environment with props and lighting (e.g. “modern living room with sofa and lamp”) around the product. Advances in neural scene synthesis (e.g. DreamSpace, Text2Scene) hint that entire layouts could be AI-filled. *Feasibility:* Experimental – would require cutting-edge AI (possibly call a neural field or diffusion model). Likely beyond our architecture until mature.  
- **Neural Mesh Editing:** Ultimate “magic eraser” and sculpting: fully replace or remove parts of the mesh via text or brush (e.g. “remove top of car” or “make chair cushions fluffier”). Emerging research (e.g. TextDeformer) uses CLIP-like guidance to deform meshes in semantic ways【34†L19-L27】【50†L1814-L1822】. *Feasibility:* Low/Research – probably not actionable yet, watch academic progress.  
- **Full Material Synthesis:** Generate physically-accurate SVBRDF textures from text or photo (e.g. convert an uploaded fabric photo into a seamless velvet material with proper roughness/normal maps). Recent work (e.g. 3DStyle-Diffusion) uses diffusion priors for fine-grained texture stylization【50†L1868-L1876】. *Feasibility:* Low – cutting-edge; would likely rely on server-side neural rendering (beyond current architecture).  
- **Immersive AR/VR Output:** Extend the viewer to push AR-ready scenes or even VR experiences (e.g. WebXR), letting business users showcase models in mobile AR or lightweight VR tours. *Feasibility:* Medium – Next.js can serve WebXR content, Firebase can host assets. The key is integration with AR frameworks (ARKit/ARCore/WebXR).  
- **AI Collaboration and Voice Control:** Integrate an AI assistant (like Spline’s voice tool) so users can say “place a spotlight on the left” or “export this to Unity” and the system executes. *Feasibility:* Experimental – prototype with speech APIs and simple commands, but complex natural-language control still nascent.  

## D) Monetization & Premium Tiers
- **Credit-Sinks:** We should gate the most compute-heavy, high-value features behind credits. Good candidates include Quad Remeshing/Retopology, high-res texture generation, style-transfer, AI background generation, and auto-rigging. These are features where users will happily spend a few credits per model. (For example, “Pro Topology (+2 credits)” for a clean mesh). Basic clean-up and standard-quality outputs remain free or low-cost.  
- **Business-Only (Paywall):** Reserve enterprise features for the 149 €/mo tier. This includes **API/Webhook export** (so companies can integrate 3D-SNAP into their systems), **White-Label Viewer** (no 3D-SNAP branding), **unlimited 4K texture exports**, **team collaboration**, **priority processing**, etc. These perks create a “premium feel” that justifies a flat monthly price. Basic users get limited credits and features; pro users pay per feature, and business users unlock these advanced integrations.  

Each proposed feature is implementable on a Next.js/Firebase stack with varying effort: lightweight features (animations, UI toggles, basic shaders) are trivial in Next.js. Heavy tasks (mesh/texture AI) would run in backend functions or external services. Overall, we can incrementally add these enhancements to position 3D-SNAP as a “studio in seconds” powerhouse【38†L54-L58】【42†L37-L45】.
